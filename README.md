# utsulog-clipflow

Personal automation tool that uploads video clips to YouTube and TikTok on a schedule.

## Overview

utsulog-clipflow runs on a Raspberry Pi 5 and processes video clips from a local pending
directory. It uploads each clip to the operator's YouTube channel and TikTok account,
records the resulting video IDs in metadata files, and backs up completed files to S3.
The tool is not distributed to or used by the general public.

## Features

- Processes pending folders in timestamp order
- Reads `post_metadata.ndjson` for per-clip title, description, and tags
- Uploads to YouTube (Data API v3) and TikTok (Content Posting API)
- Writes `yt_id` / `tt_id` back to `post_metadata.ndjson` and `ab_test_metadata.ndjson`
- Backs up completed files to S3 and removes them from local storage
- Supports separate JP / EN language runs via `--lang` flag

## Prerequisites

- Docker & Docker Compose
- YouTube OAuth credentials (`secrets/client_secrets.json`)
- TikTok access token
- AWS credentials with S3 write access

## Setup

```bash
cp .env.example .env
# Edit .env with your values
```

Place the following files in `secrets/`:

| File | Description |
|---|---|
| `client_secrets.json` | YouTube OAuth client secrets (from Google Cloud Console) |
| `token_jp.pickle` | YouTube OAuth token for JP account (generated by auth script) |
| `token_en.pickle` | YouTube OAuth token for EN account (generated by auth script) |

To generate YouTube tokens:

```bash
python scripts/auth_youtube.py --lang jp
python scripts/auth_youtube.py --lang en
```

## Usage

Run both JP and EN in sequence:

```bash
bash scripts/main.sh
```

Or run individually:

```bash
docker compose run --rm clipflow python -m clipflow.main --lang jp
docker compose run --rm clipflow python -m clipflow.main --lang en
```

Run tests:

```bash
docker compose run --rm clipflow pytest
```
